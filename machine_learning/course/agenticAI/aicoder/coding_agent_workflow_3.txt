What are the kinds of workflows you can use to when working with a coding agent.

Approach 1 and its' variations
===============================
Micro-management: You oversee what agent is doing. Reset/rewrite AGENTS.md multiple times over and re-run your workflow to compare the output from the agent.
Manually check the code generated, review and approve or refine it further.

A lot of these agents have a "Planning" mode, in which the agent prepares a kind of TODO list of tasks and design methodology to follow. So you can plan and then execute in phases.
Make sure you execute in phases. review the execution and make sure test are also written.

Finally you have a SDD: Spec Driven Development. So you trust and then finally verify.


Approach 2 and its' variations
===============================
YOLO approach -> Yolo approach means taking SDD (Spec Driven Development) to the next level and giving total autonomy to the agent to use tools, generate source code, tests, coverage, docs etc.
and then finally verify. You could give a very good AGENTS.md file and a business goal and let the agent figure it out on its own may be during an hour or two while you have dinner.

Ralph loops: This is taking YOLO to the next level by having a feedback agent also. Every time your worker agent produces some output, you feed it to the feedback agent and ask it.
1. Do you think the business goal is met?
2. Is there anything that could have been done better may be:
   a. by using latest coding standards
   b. writing more unit tests.
   c. checking performance.
   d. running valgrind
You can actually run this kind of a Ralph loop about 10 times and may be leave the agent running overnight. Then the next morning you can probably check how well it did.

An even greater level is when you have multi agent swarms (similar to stage 8 in 0_README.txt) where you have multiple agents - one for feedback, one for tests, one for coding, one for performance etc,
and they all collaborate togther to achieve the business goal. May be you have an orchestrator agent which talks to all these agents.

When yo use which workflow
==========================
If you are building Enterprise software which is mission critical, production ready, use the first approach (micro management)
But if you are building an MVP, pilot, 1st time project, and have some risk appetite, or you think the work needs a lot of HTML, boiler plate code or cookier cutter stuff. prefer the 2nd approach.
