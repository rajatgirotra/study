@ ---> Very Very useful. Claude understand @ at the claude prompt and also inside markdown files. So inside your CLAUDE.md, just put @AGENTS.md and you can use both copilot (which uses AGENTS.md) and claude together (which uses CLAUDE.md)
@ will just insert the contents of the file right there.
@ in claude prompt also shows you the internal agents that claude has.

Claude has 2 levels of granularity:

Session: (conversations you have with Claude) - You can give a name to it and also chose to resume claude with a session name.

Checkpointing: Applies to a session. Every prompt is considered a checkpoint. You can go back using rewind to go back in conversion may be to 2 chats before and also reset the code changes at that point. That is pretty cool.
Claude cannot go backwards and undo any scripts it may have run.
